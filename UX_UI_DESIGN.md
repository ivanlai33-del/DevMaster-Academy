## 🎯 設計哲學

### 核心設計原則

```yaml
以用戶為中心:
  ├─ 深入理解不同用戶的需求
  ├─ 簡化複雜流程
  ├─ 減少認知負荷
  ├─ 快速獲得價值
  └─ 持續收集用戶反饋

可訪問性優先:
  ├─ 支持多語言和文化
  ├─ 支持各種設備
  ├─ 支持各種能力用戶
  ├─ 遵循 WCAG 2.1 AA 標準
  └─ 響應式設計

效率和速度:
  ├─ 最小化點擊次數
  ├─ 快速加載時間 (< 2 秒)
  ├─ 清晰的信息架構
  ├─ 智能的默認選項
  └─ 漸進式信息披露

信任和安全:
  ├─ 透明的定價
  ├─ 明確的隱私政策
  ├─ 安全的支付流程
  ├─ 用戶數據保護
  └─ 社交證明 (評價、案例研究)
```

### 語言切換設計

#### **位置和操作方式**

```
1. 頂部導航欄
   ├─ 右上角語言選擇器
   ├─ 顯示當前語言代碼 (e.g., "EN", "中文", "日本語")
   ├─ 點擊後顯示語言下拉菜單
   ├─ 帶國旗圖標便于識別
   └─ 響應式設計：手機上作為漢堡菜單的一部分

2. 下拉菜單設計
   ├─ 按字母順序排列
   ├─ 顯示完整語言名稱和代碼
   ├─ 當前語言高亮顯示
   ├─ 搜尋功能 (用戶超過 10 種語言時)
   ├─ 動畫過渡效果
   └─ 記住用戶偏好 (LocalStorage)

3. 語言切換後的行為
   ├─ 立即應用到整個應用 (無需刷新)
   ├─ 保存用戶偏好到數據庫
   ├─ 保存到 LocalStorage 以便離線使用
   ├─ URL 不變 (後端根據用戶偏好提供語言)
   └─ 動畫過渡 (淡入淡出效果)
```

#### **用戶位置和語言自動檢測**

```yaml
首次訪問時:
  ├─ 檢測瀏覽器語言設置
  ├─ 檢測地理位置 (IP 地址)
  ├─ 智能猜測用戶偏好語言
  ├─ 顯示語言選擇建議

  ├─ 從 LocalStorage 讀取
  ├─ 從數據庫讀取 (已登入用戶)
  └─ 無縫應用
```

## 🔐 登入和身份驗證設計

### 支持的登入方式

#### **1. Email + 密碼登入**

```yaml
流程設計:
  ├─ 簡潔的登入表單
  │  ├─ Email 輸入框
  │  ├─ 密碼輸入框
  │  ├─ "保持登入" 複選框
  │  └─ 登入按鈕
  │
  ├─ 客戶端驗證
  │  ├─ Email 格式驗證 (正則表達式)
  │  ├─ 密碼長度驗證 (最少 8 個字符)
  │  ├─ 實時反饋 (綠色/紅色指示)
  │  └─ 錯誤信息清晰
  │
  ├─ 伺服器驗證
  │  ├─ SQL 注入防護
  │  ├─ 速率限制 (防止暴力破解)
  │  ├─ 密碼加密 (bcrypt)
  │  └─ 安全日誌記錄
  │
  └─ 成功登入
     ├─ JWT 令牌生成
     ├─ 令牌保存到 HttpOnly Cookie
     ├─ 自動重定向到儀表板
     └─ 記住登入狀態

UI 設計:
  ├─ 響應式表單 (手機優先)
  ├─ 清晰的輸入框標籤
  ├─ 密碼可見/隱藏切換
  ├─ 加載狀態指示
  ├─ 錯誤提示
  ├─ 成功動畫
  └─ 動畫過渡
```

#### **2. Google OAuth 登入**

```yaml
流程設計:
  ├─ 單擊 "使用 Google 登入" 按鈕
  ├─ 跳轉到 Google 登入頁面

  ├─ 返回到 DevMaster Academy
  ├─ 自動創建或登入用戶
  ├─ JWT 令牌生成
  └─ 自動重定向到儀表板

集成方式:
  ├─ 使用 Google OAuth 2.0
  ├─ 配置重定向 URI
  ├─ 處理授權代碼
  ├─ 交換訪問令牌
  ├─ 獲取用戶信息
  └─ 用戶自動同步

信息收集:
  ├─ 用戶名 (Google 名字)
  ├─ 電子郵件
  ├─ 頭像圖片
  ├─ 其他公開信息
  └─ 不收集敏感信息

UI 設計:
  ├─ Google 藍色按鈕
  ├─ 清晰的品牌標誌
  ├─ "或" 分隔符分開
  ├─ 加載動畫
  └─ 錯誤處理
```
